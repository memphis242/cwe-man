● Good call. Here's why each package is needed:

  - sqlite-devel — The runtime library (libsqlite3.so) is likely installed, but
  CMake's find_package(SQLite3) also needs the header file (sqlite3.h) to
  compile code that calls SQLite. The -devel package provides that header.
  - libcurl-devel — Same situation. libcurl.so may already be on the system
  (many tools depend on it), but curl.h is needed to compile our HTTP client
  code that calls the CWE REST API. The -devel package provides the headers.
  - nlohmann-json-devel — This is a header-only JSON library (json.hpp) that
  we'll use to parse the JSON responses from the CWE API. It's by far the
  cleanest, most idiomatic C++ JSON library. Header-only means zero linking
  complexity. There's no separate runtime — just the header.
  - ftxui — Not packaged for Fedora. We'll use CMake's FetchContent to download
  it at configure time from its GitHub repo. No system install needed.

  Shall I check which of these are already installed so we know what still needs
   to be added?
